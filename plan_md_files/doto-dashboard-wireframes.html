Parent Dashboard — Populated
/dashboard (parent session)
Seven sections, scrollable. Each section is independently collapsible by the parent (optional future enhancement). Pull-to-refresh reloads all sections in parallel.
Section order (top to bottom)

4Overdue alertNewNo overdue tasks
5Recently assigned tasksNewNo tasks assigned in last 48h

7Shopping nudgeNewList has fewer than 3 unchecked

5-day strip behaviour
5 columns: today through today+4. Today always highlighted in blue. Event dots below each date — one dot per member colour who has events that day. Max 3 dots shown; if more: "…" dot.
Tap a column → expands the event list below to show that day's events. Default: today is selected and expanded. Tap same day again → collapses.
Conflict dot: amber `#F0C070` instead of member colour. Shows even if the conflicting member has only 1 event if that event conflicts.
"+ N more → See all" if today has more than 2 events in the collapsed list. Tapping navigates to Schedule tab with that day selected.
Overdue alert
Red strip. Shows count: "2 overdue tasks". Tap "View →" navigates to Tasks tab with status filter set to "overdue". Only shown when count > 0.
Recently assigned tasks
Tasks created or updated in the last 48 hours, not yet completed. Max 3 shown. Each row: checkbox + member avatar + title + status badge. Tap checkbox → marks complete inline. "See all →" link goes to Tasks tab.
Family weekly progress
One row per family member with tasks this week. Avatar + name + "X/Y done" + progress bar in member colour. Only members who have at least 1 task this week are shown. Parents included.
Shopping nudge
Green card. Shows the list with the most unchecked items (usually Groceries). "Last updated X days ago" sub-label gives context. Only shown when the highest-count list has 3+ unchecked items. Tap "Go →" navigates to Shopping tab with that list selected.
Pending approvals nudge
Amber card per pending reward. Shows child's name, reward title, points cost, and current balance. "Review →" navigates to Rewards tab. Only shown when at least 1 reward is pending approval.

Parent Dashboard — Empty State
/dashboard (no events AND no tasks)
Shown when family has zero events AND zero tasks. The greeting changes to "Welcome, Sarah! 👋" and the subtitle becomes "Let's get your family organised". Never a blank screen.
Trigger logic
If upcomingEvents.isEmpty AND recentTasks.isEmpty → show empty variant. As soon as either has data, switch to populated variant immediately (no reload needed).
Three CTAs
Add your first event → opens AddEditEventView sheet (same as FAB)
Assign a task → opens AddEditTaskView sheet
Invite your partner → opens invite code share sheet

Child Dashboard (populated)
Child Dashboard — Populated
/dashboard (child session)
A completely separate experience from the parent dashboard. Personal, motivating, focused on the child's world. Five sections.
Header changes for child
Points balance shown right-aligned in the header (82 pts) — always visible. Avatar tap → profile settings (change password only for children).
No avatar filter row. No "+" invite button. No FAB.
Section order
#SectionHidden when
1Stats card (pts / streak / rank)Never
2Active goal progressNo active reward goal
3My tasks todayNo tasks due today
4My upcoming eventsNo events in next 3 days
5Our family member rowNever (always visible)
Stats card
Three columns: all-time pts (from points_total), streak (with grace emoji if applicable), leaderboard rank this week. Tap the rank column → navigates to Rewards tab leaderboard.
Goal progress card
Shows the child's active reward goal. Progress bar using points_balance ÷ cost. Status badge: nothing (active), "PENDING ⏳" (pending approval), "APPROVED ✓" (approved, parent to give reward). Tap → navigates to Rewards tab.
My tasks today
Only tasks where assigned_to = child's profile ID AND due_at = today. Checkable inline — tap checkbox → PATCH .../complete, points update locally immediately (optimistic). Completed tasks shown strikethrough with green pts badge.
My upcoming events
Events where child's profile ID is in assignedTo, for today + next 2 days. Max 3 shown. Conflict events in amber. Read-only — tap shows event detail sheet but no edit button.
Our family member row NEW
Horizontal scroll of all family members. Child's own avatar shown first with dark ring border and "You" label. Each member shows: avatar (32px), display name, and their weekly points or streak emoji. This is read-only — tapping a member does nothing in MVP.
Weekly points are shown for other children. For parents, show their weekly points too. Streak state shown where relevant: 🔥 for active, 🔸 for grace.

Child Dashboard — Empty State
/dashboard (child, no tasks)
Stats card is shown with zeroes — not hidden. Family members are always shown. A welcome message replaces the tasks and events sections.
Empty state rules
Stats card always shown, even at 0. This anchors the child's identity in the app from day 1.
Family members row always shown — this gives the child something to look at even before they have tasks.
Welcome card replaces tasks section: "You're in the family! Complete tasks to earn points and start your streak." Friendly, encouraging, not blank.
Greeting changes to "Hey Liam! 👋" instead of "Good morning" — more casual for a child with no data yet.

5-Day Strip
5-Day Schedule Strip
Dot legend
Dot colourMeaning
Member colourThat member has an event on this day
Amber #F0C070A conflict exists on this day
Max 3 dotsIf 4+ members have events: show 3 dots + "…" indicator
Interaction rules
Default state: Today's column is selected. Events for today shown below.
Tap any column: Selects it. Events for that day load below. The selected column gets a light blue background.
Tap same column again: Deselects. Event list collapses.
"See all in Schedule →": Navigates to Schedule tab with the selected day pre-loaded.
Member filter: If parent has tapped a member in the avatar row, event dots and expanded list filter to that member.
Data fetched
GET /api/events?from=today&to=today+4 — fetched once on load. Switching between days is client-side filtering of the already-fetched events. No additional API calls on day tap.
